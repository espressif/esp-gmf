# Multi-Source Audio Player

- [中文版](./README_CN.md)
- Example Difficulty ⭐

## Example Brief

This example implements a multi-source audio player that supports playing music from HTTP network and SD card, and can insert tone playback from Flash storage at any time. The player provides interactive control through a Command Line Interface (CLI), allowing you to switch between different audio sources, and supports operations such as play, pause, resume, and stop.

The example supports MP3, WAV, FLAC, AAC, M4A, TS, AMRNB, and AMRWB audio formats, with MP3 as the default format.

## Example Setup

### Default IDF Branch

This example supports ESP-IDF `release/v5.4` (v5.4.3 and later) and `release/v5.5` (v5.5.2 and later) branches.

### Prerequisites

This example supports three audio sources (playing flash audio will interrupt the current playback and resume the previous playback after completion):
- **HTTP Audio Source**: Play audio from network URL (default URL: `https://dl.espressif.com/dl/audio/ff-16b-2c-44100hz.mp3`)
- **SD Card Audio Source**: Play audio from SD card file (default path: `/sdcard/test.mp3`)
- **Flash Embedded Audio**: Play audio files embedded in Flash at compile time (related path: `embed://tone/0_alarm.mp3`)

The audio files used for flash in this example are in the [Embedding Binary Data](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/api-guides/build-system.html#cmake-embed-data), which is compiled with the code and downloaded to the flash memory.
The `esp_embed_tone.h` and `esp_embed_tone.cmake` files for Flash embedded audio are generated by the script `gmf_io/mk_flash_embed_tone.py`. If you need to replace the audio files, you must rerun the script to regenerate these two files. Python script command is as follows:

```
python ../../../elements/gmf_io/mk_flash_embed_tone.py -p components/tone/
```

### Build and Flash

Before compiling this example, ensure that the ESP-IDF environment is properly configured. If it is already set up, you can proceed to the next configuration step. If not, run the following script in the root directory of ESP-IDF to set up the build environment. For detailed steps on configuring and using ESP-IDF, please refer to the [ESP-IDF Programming Guide](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/index.html)

```
./install.sh
. ./export.sh
```

Here are the summarized steps for compilation:

- Enter the location where the multi-source audio player project is stored

```
cd gmf_examples/basic_examples/pipeline_play_multi_source_music
```

- Execute the prebuild script, follow the prompts to select the target chip, automatically setup IDF Action Extension

On Linux / macOS, run following command:
```bash/zsh
source prebuild.sh
```

On Windows, run following command:
```powershell
.\prebuild.ps1
```

- Configure Wi-Fi SSID/password (needed for HTTP playback)

Configure `WiFi SSID` and `WiFi Password` via `idf.py menuconfig` -> `GMF APP Configuration` -> `Example Connection Configuration`. If the Wi-Fi configuration is incorrect, the device will keep reconnecting and HTTP online playback will not work properly.

- Build the Example

```
idf.py build
```

- Flash the program and run the monitor tool to view serial output (replace PORT with the port name):

```
idf.py -p PORT flash monitor
```

- Exit the debugging interface using ``Ctrl-]``

## How to use the Example

### Functionality and Usage

After the example starts, it will:
1. Initialize WiFi connection (for HTTP audio source)
2. Initialize SD card (for SD card audio source)
3. Create audio pipeline and register CLI commands
4. Start playing audio from SD card by default

After startup, you can use the following CLI commands in the serial terminal to control the player:

- `play` - Start playback from current audio source
- `pause` - Pause playback
- `resume` - Resume playback
- `stop` - Stop playback
- `switch [http|sdcard]` - Switch audio source (toggles between HTTP and SD card if no argument provided)
- `get_vol` - Get current volume (0-100)
- `set_vol <0-100>` - Set volume (0-100)
- `status` - Show current playback status
- `tone` - Play embedded audio from Flash (pauses current playback, plays tone, then automatically resumes)
- `exit` - Exit the application
- `help` - Show all available commands
