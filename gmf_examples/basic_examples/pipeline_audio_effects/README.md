# Audio Effects Playback Demo

- [中文版](./README_CN.md)
- Example Difficulty ⭐

## Overview

This demo is built on the Espressif Generic Media Framework (GMF) and shows how to assemble and run multiple audio-processing pipelines. With the embedded audio assets you can quickly experience different GMF audio effects and a mixer workflow, helping you understand both the API usage and the audible results.

- **Audio-effects showcase**: adjust effect parameters on the fly (for example EQ gain or DRC compression curves) while the track is playing and hear the change immediately.
- **Mixer showcase**: demonstrate real-time control of multiple audio sources by layering a prompt tone on top of background music and applying fade-in/out behaviour.

## Environment Requirements

### ESP-IDF Version

This example supports ESP-IDF `release/v5.4` (v5.4.3 and later) and `release/v5.5` (v5.5.2 and later) branches.

### Prerequisites

The audio clips are provided as [embedded binary data](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/api-guides/build-system.html#cmake-embed-data), so they are compiled into flash together with the code.

Two sample files, `manloud_48000_2_16_10.wav` and `tone.mp3`, live in `${PROJECT_DIR}/components/music_src/`. The files `esp_embed_tone.h` and `esp_embed_tone.cmake` are generated by `$YOUR_GMF_PATH/elements/gmf_io/mk_flash_embed_tone.py`. If you need to replace the audio assets, rerun the script to regenerate those two files:

```
python $YOUR_GMF_PATH/elements/gmf_io/mk_flash_embed_tone.py -p $YOUR_GMF_PATH/gmf_examples/basic_examples/pipeline_audio_effects/components/music_src
```

### Configuration

- **Effect Demo**: All audio effect demos (ALC, Sonic, EQ, Fade, DRC, MBC, and Mixer) are enabled by default. Users can select the effects they are interested in for demonstration in the "Pipeline Audio Effects Example" menu in menuconfig.
- **Audio Type**: The audio files used in this example are in WAV and MP3 formats, and the corresponding decoders are registered by default. To use audio files in other formats, please go to "Audio Codec Configuration" → "Audio Decoder Configuration" or "Audio Simple Decoder Configuration" in menuconfig and enable the support options for the corresponding audio decoders.

### Build and Flash

Before compiling this example, ensure that the ESP-IDF environment is properly configured. If it is already set up, you can proceed to the next configuration step. If not, run the following script in the root directory of ESP-IDF to set up the build environment. For detailed steps on configuring and using ESP-IDF, please refer to the [ESP-IDF Programming Guide](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/index.html):

```
./install.sh
. ./export.sh
```

Here are the summarized steps for compilation:

- Enter the location where the audio effects test project is stored

```
cd $YOUR_GMF_PATH/gmf_examples/basic_examples/pipeline_audio_effects
```

- Execute the prebuild script, select the target chip, automatically setup IDF Action Extension

On Linux / macOS, run following command:
```bash/zsh
source prebuild.sh
```

On Windows, run following command:
```powershell
.\prebuild.ps1
```

- Use `esp_board_manager` to select supported board and custom board, be sure to see [ESP Board Manager](https://components.espressif.com/components/espressif/esp_board_manager), taking ESP32-S3-Korvo2 V3.1 as an example:

```
idf.py gen-bmgr-config -b esp32_s3_korvo2_v3
```

- Build the Example

```
idf.py build
```

- Flash the program and run the monitor tool to view serial output (replace PORT with the port name):

```
idf.py -p PORT flash monitor
```

- Exit the debugging interface using ``Ctrl-]``

## Usage Instructions

### Function Description

- **Single-effect mode**
  1. Reads and decodes the embedded WAV file `manloud_48000_2_16_10.wav` from flash.
  2. Plays the first four seconds with neutral parameters.
  3. After the delay the demo calls the relevant APIs to update effect parameters and highlight the change.
  4. When playback ends the pipeline stops automatically and releases GMF resources.

- **Mixer mode**
  1. Pipeline A decodes the background music `manloud_48000_2_16_10.wav` from flash.
  2. Pipeline B starts later, decoding the prompt tone `tone.mp3` and automatically converting sample rate, channel count, and bit depth.
  3. Pipeline C mixes both sources and applies fade control based on tone events.
  4. Once all inputs finish, the mixer and its pipelines exit and free their resources.
