The audio files used in this example are in [embedded binary](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/api-guides/build-system.html#cmake-embed-data) format and are compiled and downloaded to flash with the firmware.

This example provides two test files, `manloud_48000_2_16_10.wav` and `tone.mp3`, in the `${PROJECT_DIR}/components/music_src/` folder. The files `esp_embed_tone.h` and `esp_embed_tone.cmake` are generated by `$YOUR_GMF_PATH/elements/gmf_io/mk_flash_embed_tone.py`. To use different audio files, run the script to regenerate them:

```
python $YOUR_GMF_PATH/elements/gmf_io/mk_flash_embed_tone.py -p $YOUR_GMF_PATH/gmf_examples/basic_examples/pipeline_audio_effects/components/music_src
```

### Audio file not found

If you see the following error in the log, the audio file to play was not found on the microSD card. Ensure the file is named correctly and stored on the card:

```c
E (1133) ESP_GMF_FILE: Failed to open on read, path: /sdcard/test.mp3, err: No such file or directory
E (1140) ESP_GMF_IO: esp_gmf_io_open(71): esp_gmf_io_open failed
```
